# 增强模块设计文档

## 概述

此技能通过分析实际实现代码来增强现有的模块设计文档。它填补缺失的细节，修正过时的信息，提供更准确和完整的设计文档。

## 使用场景

- 用户有现有但不完整/过时的设计文档
- 代码已实现但文档未更新
- 需要用更好的设计文档来记录遗留代码
- 用户想要用实际代码分析来补充粗糙的设计文档

## 工作流程

### 第一步：收集用户输入

1. **设计文档**：请用户提供原始设计文档
   - 支持格式：PDF、DOCX、Markdown
   - 如果是 PDF：使用 pdf-processing 技能提取内容
   - 如果是 DOCX：使用 docx-processing 技能提取内容
   - 如果是 Markdown：直接读取

2. **代码仓库**：请用户选择代码仓库路径
   - 必须是有效的本地路径
   - 验证路径存在且包含代码

3. **输出目录**：请用户指定保存增强文档的位置
   - 默认为项目根目录下的 `docs/` 文件夹

### 第二步：查找入口点

1. 分析设计文档以了解功能范围
2. 在代码仓库中搜索相关入口点：
   - 文档中提到的主函数/类
   - API 端点、控制器、处理器
   - 服务类、业务逻辑模块
3. 如果未找到入口点：请用户指定
4. 如果找到多个入口点：向用户展示选项供其选择

### 第三步：选择模板

向用户展示 4 种模板选项：

| 模板 | 侧重点 |
|------|--------|
| **普通功能类** | 基础功能、业务逻辑、接口定义 |
| **管理系统类** | CRUD 操作、数据流、权限控制、状态管理 |
| **算法类** | 算法流程、时间/空间复杂度、边界情况 |
| **中间件类** | 底层通信、协议处理、性能优化、容错机制 |

用户也可以选择自定义或提供自己的模板。

### 第四步：分析代码和文档

使用 Claude API 进行分析：

1. **业务/算法流程**
   - 将主要业务流程拆解为子流程
   - 识别关键模块及其交互
   - 生成 Mermaid 流程图和时序图

2. **数据模型**
   - 输入/输出数据结构
   - 数据库表结构（如果有）
   - 关键字段和关系
   - 生成 Mermaid ERD 图

3. **类/模块结构**
   - 主要类及其职责
   - 包结构
   - 依赖关系
   - 生成 Mermaid 类图

4. **与原始文档对比**
   - 识别缺失的章节
   - 识别过时的信息
   - 识别错误的描述

### 第五步：生成增强文档

生成 Markdown 文档，包含：

1. 原始内容经代码分析增强后
2. Mermaid 图表：
   - 流程图（业务流程）
   - 时序图（时序图）
   - 类图（类图）
   - ERD 图（数据库表结构）
3. 基于所选模板的清晰章节结构
4. 末尾的版本历史表格

### 第六步：模板自我增强

生成文档后，询问用户：

> "是否启用模板自我增强？这将帮助根据本次执行结果改进模板。"

如果用户同意：
1. 展示生成的文档供用户审阅
2. 请用户确认或修改具体部分
3. 根据用户反馈更新模板

## 关键原则

1. **始终分析实际代码** - 不要仅依赖原始文档
2. **生成 Mermaid 图表** - 可视化表示对理解至关重要
3. **分解大型流程** - 将复杂业务流程拆解为可管理的子流程
4. **保留原始内容** - 增强而非替换
5. **请求用户确认** - 特别是关键部分
6. **支持自我增强** - 根据用户反馈改进模板
