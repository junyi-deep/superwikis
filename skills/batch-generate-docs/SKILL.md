# 批量生成完整设计文档

## 概述

此技能结合了增强模块设计和生成项目总览的功能，可为整个项目生成完整的设计文档集。它处理多个模块，生成各个增强的模块文档和项目总览文档。

## 使用场景

- 需要为包含多个模块的整个项目编写文档
- 一次性处理多个模块
- 启动全面的文档工作
- 更新项目中的所有模块文档

## 工作流程

### 第一步：收集用户输入

1. **模块设计文档目录**：请用户提供：
   - 包含多个模块设计文档的目录
   - 每个文件可以是 PDF、DOCX 或 Markdown
   - 验证目录存在且包含设计文档

2. **代码仓库**：请用户选择代码仓库路径
   - 必须是有效的本地路径
   - 包含所有要记录的模块

3. **输出目录**：请用户指定保存所有文档的位置
   - 默认为项目根目录下的 `docs/`
   - 如果未指定，创建 `docs/` 目录

4. **模板配置**：
   - 询问是否所有模块使用相同模板
   - 或让用户为每个模块指定模板
   - 选项：普通功能类、管理系统类、算法类、中间件类、自定义

### 第二步：处理每个模块

对于找到的每个模块设计文档：

1. **解析原始文档**
   - 提取模块名称
   - 提取现有设计细节

2. **在代码中查找模块**
   - 搜索入口点
   - 识别模块边界
   - 如果未找到，请用户指定

3. **选择合适的模板**
   - 使用用户指定的模板
   - 或让用户确认每个模块

4. **分析代码**
   - 使用 Claude API 分析：
     - 业务流
     - 数据模型
     - 类结构
     - 依赖关系

5. **生成增强文档**
   - 创建带 Mermaid 图表的 Markdown
   - 应用选定的模板结构

### 第三步：生成项目总览

处理完所有模块后：

1. **分析模块关系**
   - 识别模块之间的依赖
   - 识别共享组件

2. **生成架构图**
   - 使用 Mermaid C4 图
   - 展示系统上下文
   - 展示容器关系

3. **创建模块索引**
   - 所有模块的表格
   - 详细文档的链接

4. **生成总览文档**
   - 项目总览
   - 技术栈
   - 模块列表
   - 依赖关系

### 第四步：模板自我增强

生成所有文档后，询问用户：

> "是否启用模板自我增强？这将帮助根据本次批量执行结果改进模板。"

如果用户同意：
1. 展示生成的文档摘要
2. 请用户审阅关键部分
3. 将反馈应用于模板

## 输出结构

```
docs/
├── index.md                          # 项目总览文档
├── module-index.md                   # 模块索引
└── modules/
    ├── module-a-design.md           # 增强后的模块 A 设计
    ├── module-b-design.md           # 增强后的模块 B 设计
    └── module-c-design.md           # 增强后的模块 C 设计
```

## 关键原则

1. **系统化处理** - 一致地处理每个模块
2. **生成全面的图表** - 每个模块都有流程图、时序图、类图、ERD 图
3. **展示关系** - 清晰可视化模块之间的依赖
4. **创建导航** - 便于在文档之间查找和导航
5. **批量模板自我增强** - 根据批量执行改进模板
6. **验证输出结构** - 确保所有文件都正确创建
